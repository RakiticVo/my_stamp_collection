# Context & Planning Workflow (CRITICAL)

Always use this workflow when receiving a new task or if a task becomes ambiguous.

## 1. Context & Planning Rules
Refer to the **Core Planning Rules**: [core.md](file:///c:/Work/FlutterProjects/my_stamp_collection/.agent/rules/core.md#rule-0-planning--approval-first-mandatory) and [workflow_planning.md](file:///c:/Work/FlutterProjects/my_stamp_collection/.agent/rules/workflow_planning.md)

## 2. Execution Steps
1. **Analyze for Ambiguity**: Identify files, required changes, dependencies, and existing patterns.
2. **Ask Clarifying Questions (if needed)**: Ask specific questions to fill gaps.
3. **Present Proposed Plan**: Provide a concise but detailed plan with files and steps.
4. **Obtain Explicit Approval**: Ask: *"Does this plan look correct to you? Once you approve, I will begin."*
5. **STOP**: Do NOT proceed until the USER provides permission.
